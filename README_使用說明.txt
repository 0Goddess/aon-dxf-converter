Project XML 轉 AON DXF 1.8.0-TEST4
=============================

用途
----
將 Microsoft Project 匯出的 XML 轉成 AON 全區網圖 DXF。
輸出檔可使用 AutoCAD 2023 開啟；DXF 採 R2010/AC1024 格式。

使用方式
--------
1. 解壓縮完整程式包，不要只複製 EXE。
2. 執行 AON_XML_to_DXF.exe。
3. 選擇 Microsoft Project XML。
4. 選擇時間座標與分區方式。
5. 按「開始轉換」；DXF 預設輸出在 XML 所在資料夾。

輸出內容
--------
- 全區 AON 網圖 DXF
- 節點顯示 ID、WBS、作業名稱、ES、D、EF、LS、TF、FF、LF
- 紅色：Project 要徑作業及要徑關係
- 關係線：單一 LWPOLYLINE 物件，附前後作業 XDATA
- 出圖前會再調整非要徑作業列，以減少箭線交錯、上下折返及轉角
- 完整 FS 作業鏈可整串上下移動，並以全圖箭線交錯最少為最高原則
- TEST：多後續分岔不受同分條件限制，後續總作業較多的路線優先成為水平主線
- TEST：時間順序、分區與作業順序改為次要排序；箭線交錯、作業框避碰與路由硬規則仍保留
- TEST3：新版面封閉出口時，依序測試水平空白欄與垂直空白列，兩方向各最多增加兩格
- TEST3：水平、垂直擴距仍無解時，回復穩定主線版面，不放寬任何箭線硬規則
- TEST4：要徑固定中央水平，非要徑以工作框數量最多的連續鏈作為水平支幹
- TEST4：長支幹優先靠近所屬主幹，其他完整分支向上下展開；時段重疊時自動插入作業列
- TEST4：工作框排版完成後鎖定，再交由既有箭線規則繪製
- 主水平鏈不因額外前置／後續分支而拆開；區域內完整作業列可交換上下順序
- 單一短分支優先配置在所屬鏈附近，但不得增加箭線交錯
- 安全候選中只要存在零交錯路徑，就禁止採用有交錯路徑
- 同一來源的全部後續依目標實際高度由上往下配置出口，水平關係不強制置中
- 水平直線與繞線共用同一份來源出口順位，後續端點配置不得再次覆寫
- 向上分支由最高目標起使用最近轉角；向下分支採鏡射順位，由最低目標起使用最近轉角
- 出口、第一轉角與垂直通道順位完成後即鎖定，後處理不得交換
- 同一時間欄的相鄰多後續來源使用互不穿插的專屬垂直通道帶
- 任何後處理候選若新增正交交錯即撤銷，不得破壞已完成的零交錯扇出
- 固定局部通道不可用時先掃描所有列間空白高度，最後才允許全圖外框繞行
- 路由與後處理會計算X方向反轉，存在單調候選時禁止矩形迴繞
- 路由同時計算Y方向反轉，禁止超過目標高度後再折返回到目標
- 安全的水平－垂直－水平路徑存在時，禁止使用多餘階梯轉角
- 相鄰來源若後續通道範圍重疊，會合併比較扇出順位，不再只依相同起點座標分組
- 向上分支固定為目標越高、第一個垂直轉角越靠近來源，後處理不得改變此順位
- 跨多列長關係優先使用獨立空白通道，避免混入端點附近的局部扇出
- 非端點作業框外保留 12 單位硬性路由安全區
- 排版評分採正交 H-V-H 路徑估算，長垂直通道穿越水平作業鏈會優先消除
- 非直線關係的起點與終點固定保留至少 28 單位水平緩衝
- 完成路由後再次分離過近的平行線與轉角群組
- 可連續且時段不衝突的 FS 作業，優先排成同一水平列
- 同一作業的多條後置關係依目標上下順序分開出口與通道
- 相鄰作業的短關係優先保留水平直線，不讓長距離關係迫使它 U 型外繞
- 跨列關係優先採「水平－垂直－水平」兩轉折
- 無法避開作業框時才使用上／下通道外繞
- 中文轉成向量線條，因此不依賴使用者電腦中文字型

重要說明
--------
- ES、EF、LS、LF、TF、FF 以 Project XML 內的計算結果為準。
- 本程式不修改原始 XML，也不需要安裝 Microsoft Project。
- 產生 DXF 不需要安裝 AutoCAD；AutoCAD 只用於開啟成果。
- 轉換大型排程可能需要數十秒，中文字向量化期間請勿關閉程式。
- 程式未購買商業程式碼簽章時，Windows 可能顯示 SmartScreen 提示。

從原始碼建立 Windows 免安裝版
------------------------------
1. 在 Windows 64 位元電腦安裝 Python 3.12。
2. 執行 build_windows.cmd。
3. 完成檔位於 dist\AON_XML_to_DXF_Windows64.zip。
