Project XML 轉 AON DXF 1.0
===========================

用途
----
將 Microsoft Project 匯出的 XML 轉成 AON 全區網圖 DXF。
輸出檔可使用 AutoCAD 2023 開啟；DXF 採 R2010/AC1024 格式。

使用方式
--------
1. 解壓縮完整程式包，不要只複製 EXE。
2. 執行 AON_XML_to_DXF.exe。
3. 選擇 Microsoft Project XML。
4. 選擇輸出資料夾。
5. 按「開始轉換」。

輸出內容
--------
- 全區 AON 網圖 DXF
- 節點顯示 ID、WBS、作業名稱、ES、D、EF、LS、TF、FF、LF
- 分區名稱由每份 XML 的 WBS／摘要作業自動辨識，不固定套用 A1、A2、B
- 上方顯示月份時間座標，作業依主要占用月份排列；跨月作業以涵蓋天數最多的月份決定位置
- 時間分隔可選自動、週、月、季、年或關閉；跨期間作業以涵蓋天數最多的期間決定位置
- 關係箭線採較深且可區分的 FS、SS、FF、SF 色彩；要徑維持深紅色
- 分區方式可選自動、不分區或 WBS 第1／2／3層；以該層摘要作業名稱作為分區
- 要徑固定於網圖中央水平帶；同時間段的要徑作業使用不同水平槽位，不互相重疊
- 非要徑作業置於中央要徑的上下方；保留水平鏈減少彎折，作業較少的分區／列置上
- 箭線路由具上／下外框保底通道；極端密集關係不再因 No node-clear route 中止轉換
- 多後置作業使用分散的節點端口；水平、垂直及外繞通道均提高間距，避免箭線共線重疊
- 分區內以關係重心反覆排序作業列，將相互關聯節點拉近以減少線段交錯
- 全圖平行線淨距稽核：垂直至少 14、水平至少 9 單位；不足即改走獨立通道
- 箭線使用醒目的 CAD 真彩色並增加線寬：FS 淡藍、SS 青藍、FF 亮綠、SF 洋紅、要徑鮮紅
- DXF 預設輸出到匯入 XML 的同一資料夾，不另建輸出資料夾
- 紅色：Project 要徑作業及要徑關係
- 關係線：單一 LWPOLYLINE 物件，附前後作業 XDATA
- 跨列關係優先採「水平－垂直－水平」兩轉折
- 無法避開作業框時才使用上／下通道外繞
- 中文轉成向量線條，因此不依賴使用者電腦中文字型

重要說明
--------
- ES、EF、LS、LF、TF、FF 以 Project XML 內的計算結果為準。
- 本程式不修改原始 XML，也不需要安裝 Microsoft Project。
- 產生 DXF 不需要安裝 AutoCAD；AutoCAD 只用於開啟成果。
- 轉換大型排程可能需要數十秒，中文字向量化期間請勿關閉程式。
- 程式未購買商業程式碼簽章時，Windows 可能顯示 SmartScreen 提示。

從原始碼建立 Windows 免安裝版
------------------------------
1. 在 Windows 64 位元電腦安裝 Python 3.12。
2. 執行 build_windows.cmd。
3. 完成檔位於 dist\AON_XML_to_DXF_Windows64.zip。
